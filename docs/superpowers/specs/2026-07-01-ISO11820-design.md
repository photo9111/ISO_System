## 三、核心服务设计

### 3.1 SensorSimulator — 仿真引擎
- 纯算法类，持有 `Random` 实例
- `Update(currentTemps, state, config)` → 新温度字典
- 四个分支：升温 / 稳定 / 记录 / 降温
- 5 通道：TF1、TF2、TS、TC、TCal

### 3.2 DaqWorker — 数据采集
- `System.Timers.Timer` 每 800ms Tick
- 仿真模式调用 `SensorSimulator.Update()`
- 调用 `TestController.DoWork()` 推进状态机
- 构建 `DataBroadcastEventArgs` 触发事件

### 3.3 TestController — 状态机
5 状态：Idle → Preparing → Ready → Recording → Complete
- `DoWork()`：稳定判定、终止条件检查、计时器、消息生成
- 按钮方法：StartHeating / StopHeating / StartRecording / StopRecording
- 完整状态流转见开发文档 2.4 节

### 3.4 ExportService
- CSV：StreamWriter 原生写入
- Excel：EPPlus，3 个 Sheet（试验信息 / 温度数据 / 曲线图）
- PDF：PDFsharp-MigraDoc

### 3.5 AppContext — 全局单例
持有 DbHelper、TestController、DaqWorker、当前操作员信息

## 四、数据流

```
DaqWorker (后台线程, 800ms)
  ├→ SensorSimulator.Update()
  ├→ TestController.DoWork()
  └→ DataBroadcast 事件 → MainForm.Invoke() → UI 更新
```

## 五、构建顺序

按分层自底向上：

1. **项目骨架**：sln + csproj + Program.cs + appsettings.json
2. **配置读取**：Microsoft.Extensions.Configuration 加载 appsettings.json
3. **数据层**：Models + DbHelper + DbInitializer
4. **仿真引擎**：SensorSimulator + DriftCalculator
5. **服务层**：DaqWorker + ExportService
6. **业务核心**：TestController（状态机）
7. **全局层**：AppContext
8. **UI 层**：LoginForm → MainForm（温度显示+曲线+消息日志+按钮控制）→ NewTestForm → TestRecordForm → CalibrationForm
9. **记录查询**：MainForm 中的查询 Tab
10. **集成调试**：跑通完整 9 步演示流程
